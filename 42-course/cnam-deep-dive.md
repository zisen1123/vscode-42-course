# CNAM 文档深度讲解（IoT + NFV）

## 1) Iot_MasterInternational_2022_2023.pdf（深度讲解）

定位：这是一个 IoT 课程总览文档，课程结构明确分成 4 个主题 + 4 次实验。

核心知识主线：
- IoT 定义与术语：设备、传感器、网关、云平台、应用层。
- IoT 架构：感知层、网络层、平台层、应用层。
- 传感器网络通信：短距/长距技术对比（覆盖、功耗、吞吐、成本）。
- IoT 中机器学习：分类与回归在定位、识别任务中的应用。
- IoT 定位：动机、算法分类、实验设计。

从课程结构提炼的学习动作：
- 先画架构图：把“设备->传输->平台->业务”画成单向数据流。
- 再做通信技术对照表：如 Wi-Fi/BLE/Zigbee/LoRa 的覆盖、功耗、带宽。
- 最后把 ML 任务拆成标准流程：数据采集 -> 特征 -> 训练 -> 评估 -> 部署。

容易混淆的点：
- 分类 vs 回归：定位任务可转成分类（区域）也可做回归（坐标）。
- “能连上网”不等于“系统可运维”：还要考虑设备管理、版本升级、故障监控。

可执行练习：
- 练习 A：为“室内定位”写一个最小系统设计说明（输入、输出、指标）。
- 练习 B：给 3 种无线技术做选型结论（为什么选、为什么不选）。

## 2) Understanding NFV basics NEVA 2022-2023.pdf（深度讲解）

定位：NFV 概念与技术基础教程，强调 ETSI 视角下的架构、收益、挑战。

核心知识主线：
- NFV 概念：把网络功能从专用硬件迁移到通用服务器资源池。
- 关键对象：PNF、VNF、网络服务（NS）、MANO（管理与编排）。
- 目标收益：降 CAPEX/OPEX、提高弹性、加快服务上线、支持多租户与切片。
- 技术挑战：性能隔离、编排复杂度、生命周期管理、可观测性与可靠性。

架构理解顺序（建议）：
- 第一步：明确“功能”和“基础设施”分离（软件化）。
- 第二步：理解 MANO 在部署、扩缩容、升级中的职责。
- 第三步：把 5G slice 看作多个网络服务实例在共享基础设施上的编排结果。

容易混淆的点：
- NFV != SDN：NFV偏“功能虚拟化”，SDN偏“控制与转发分离”；实践中常结合。
- 虚拟化后并非自动高性能：还要做资源调度、亲和性、加速与观测。

可执行练习：
- 练习 A：画一张“传统专用设备网络”到“NFV 网络”的迁移前后图。
- 练习 B：定义一个 VNF 生命周期（创建、扩容、升级、回滚、退役）状态机。
